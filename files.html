File Uploads Project

Overview

This project consisted of redesigning a specific functionality within internal enterprise software. In order to protect the privacy of the product and organzation, I will focus on the specific problem and solution, leaving out the "bigger picture".

Uploading files within the software is outdated and inefficient. Additionally, it does not take into account the varied needs of users. I was tasked to research the use of more up-to-date technologies, such as Bootstrap, to redesign and improve the experience of uploading documents to one central system.

Considerations

  • Multiple types of users in different scenarios
  • Some users could have spotty internet connections
  • Different age demographics
  • Different file extension types
  • Multiple file uploads should be available when necessary

Edited screenshot of the original upload method to be redesigned based on user needs

I created multiple task flows to demonstrate the user's interactions with the upload files function.

  • The first task flow shown below is for simple, single file uploads. This single file uploads task flow was created with two different options for uploading files: option #1 consisted of an "Add File" button that, when clicked on, allows the user to add a file through the more commonly established file dialog box. Option #2 allows the user to drag and drop to add a file. This allows for flexibility within the users age demographic and familiarity with newer methods of uploads.

    • Task Flow #1

  • The second task flow was created for multiple file uploads and allows for two different methods of uploading as well. This task flow is similar to the one for single file uploads, with the main difference being that the user can upload multiple files at once.

    • Task Flow #2

  • Additionally, I visualized error handling scenarios to remove further frustrations from any errors the user might commit. These error scenarios used inline validation as much as possible.They are also meant to be as specific as possible to further alleviate guesswork; with the error message detailing what the error is and how to fix it.
